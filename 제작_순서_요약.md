# IBS 법률 AI 시스템 제작 순서 요약

## 📋 프로젝트 개요
RAG 기반 법률 정보 제공 시스템 구축

## 🗂️ RAG DB 구성 요소 (10가지)
1. 법령 데이터 (statute)
2. 판례 요약 데이터 (case)
3. 절차 매뉴얼 (procedure)
4. 실무 매뉴얼 (manual)
5. 사건 유형 정의서 (case_type)
6. 사건별 템플릿 구조 (template)
7. 양형기준 요약 (sentencing_guideline)
8. 법률 FAQ (faq)
9. 키워드-사건 맵핑 (keyword_mapping)
10. 스타일 문제 데이터 (style_issue)

## 🚀 개발 단계 (10단계)

### Phase 1: 프로젝트 기반 구조 설계 (1주)
- 프로젝트 디렉토리 구조 생성 ✅
- 개발 환경 설정
- README 및 기본 문서 작성 ✅

### Phase 2: 데이터 모델 및 스키마 정의 (1주)
- 공통 BaseDocument 모델
- 10가지 데이터 타입별 Pydantic 모델
- JSON 스키마 검증 도구

### Phase 3: 데이터 수집 및 변환 도구 (2주)
- 데이터 수집 스크립트
- JSON 변환 및 정제 도구
- 샘플 데이터 생성

### Phase 4: 벡터 DB 설정 및 임베딩 (1-2주)
- 벡터 DB 선택 및 설정 (Chroma/Pinecone)
- 임베딩 모델 설정 (한국어 최적화)
- 데이터 인덱싱 파이프라인

### Phase 5: RAG 시스템 핵심 로직 (2-3주)
- LangGraph 워크플로우 설계
- 검색 및 검색 강화 모듈
- 프롬프트 엔지니어링
- LLM 통합

### Phase 6: API 서버 개발 (1-2주)
- FastAPI 서버 구축
- 검색/질의응답 API
- 인증 및 보안

### Phase 7: 고급 기능 개발 (2주)
- 멀티 턴 대화 관리
- 검색 결과 후처리
- 키워드-사건 맵핑 활용
- 스타일 문제 검증

### Phase 8: 테스트 및 검증 (1-2주)
- 단위/통합 테스트
- 성능 테스트
- 정확도 검증

### Phase 9: 모니터링 및 로깅 (1주)
- 로깅 시스템
- 모니터링 대시보드
- 알림 시스템

### Phase 10: 문서화 및 배포 (1주)
- API 문서화
- 배포 준비 (Docker)
- CI/CD 파이프라인

## ⏱️ 예상 기간
- **MVP**: 6-8주
- **전체 시스템**: 12-16주

## 🎯 우선순위
1. **최우선 (MVP)**: Phase 1-6, 8 (기본 기능)
2. **2차 우선**: Phase 3 (전체 데이터), Phase 5 (고급 기능), Phase 7
3. **3차 우선**: Phase 9-10 (운영 및 배포)
